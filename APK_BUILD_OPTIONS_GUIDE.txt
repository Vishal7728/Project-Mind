OPTION 1: WSL2 (BEST PERFORMANCE - RECOMMENDED)
════════════════════════════════════════════════════════════════════════════════

Requirements: Windows 10/11 Build 18917+, Hyper-V capable
Installation time: ~5-10 minutes
Build time: 15-30 minutes (first build is slower)

STEP 1: Install WSL2
────────────────────────────────────────────────────────────────────────────────
1. Right-click PowerShell → "Run as Administrator"
2. Paste this command:

   wsl --install -d Ubuntu

3. Press Enter and wait (may take 5-10 minutes)
4. RESTART YOUR COMPUTER
5. Ubuntu will finish setup on next boot (complete setup prompts)

STEP 2: Build APK
────────────────────────────────────────────────────────────────────────────────
1. Open PowerShell (normal, non-admin)
2. Type: wsl
3. This opens Ubuntu terminal. Paste this entire block:

   sudo apt-get update && \
   sudo apt-get upgrade -y && \
   sudo apt-get install -y python3 python3-pip openjdk-11-jdk-headless build-essential git libssl-dev libffi-dev libxml2-dev libxslt1-dev libjpeg-dev zlib1g-dev && \
   pip install buildozer Cython==0.29.33 virtualenv && \
   pip install kivy Pillow && \
   cd /mnt/d/Project\ Mind && \
   buildozer android release

4. Wait 15-30 minutes for completion
5. APK at: d:\Project Mind\bin\projectmind-1.0.0-release.apk

ADVANTAGES:
✅ Native Linux build (most reliable)
✅ Fastest subsequent builds
✅ Full control and debugging
✅ Works on any machine with Windows 10/11

TROUBLESHOOTING
════════════════════════════════════════════════════════════════════════════════

"wsl: The term 'wsl' is not recognized"
Solution: Update Windows or enable WSL2 manually

"Error: Virtual machine failed"
Solution: Enable Hyper-V in Windows
Windows 10 Pro: Ctrl+R → optionalfeatures → ✓ Hyper-V
Windows 11: Settings → System → Optional features → Hyper-V

"Docker: command not found"
Solution: Install Docker Desktop from docker.com

"Build hangs or times out"
Solution: 
- First build is slow (downloads SDK, dependencies)
- Subsequent builds are faster
- Ensure 10-15GB free disk space
- Check internet connection

"APK not created"
Solution:
- Check build output for errors
- Ensure buildozer.spec is valid
- Check disk space
- Try: buildozer android clean first
